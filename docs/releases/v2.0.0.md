# GTD Space {{tag}}

## Highlights

- Added a GTD Space MCP server with a generated context pack so local model clients can inspect and operate on a workspace through semantic GTD resources and tools instead of raw file editing.
- Refactored the GTD markdown domain logic for habits, projects, actions, and metadata parsing to tighten compatibility with the workspace spec and reduce edge-case drift.
- Landed major frontend architecture cleanup across tab management, sidebar behavior, and BlockNote preprocessing to make the app steadier during day-to-day editing.

## Notable Changes

- MCP resources now expose workspace context, GTD schema guidance, and structured item summaries, while mutation flows use dry-run planning plus explicit apply for safer automation.
- Habit, project, and action handling now share more consistent parsing and domain logic, with better behavior around dates, history, metadata extraction, and legacy markdown compatibility.
- Recent UI refactors improved sidebar resilience, tab lifecycle handling, and editor preprocessing without changing the file-backed nature of GTD Space.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- First-time users: GTD Space creates the default workspace at `~/GTD Space`
- Local-model integrations can now target GTD Space through the new MCP server over stdio
- Existing GTD workspaces remain markdown-first and compatible with the app’s on-disk structure
