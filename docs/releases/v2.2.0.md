# GTD Space {{tag}}

## Highlights

- Added MCP habit-history tooling so integrations can replace and manage canonical habit history more directly.
- Hardened settings persistence and sidebar reload behavior to avoid broken refresh paths when alias or workspace state is incomplete.

## Notable Changes

- GTD Space now seeds canonical habit history data and rejects legacy rows that do not match the new history model.
- The MCP server gained tools for habit history replacement workflows, giving automation a safer path for bulk history updates.
- Settings save behavior and workspace sidebar reloads are more defensive, especially when optional alias sections or partial state are missing.
- Rust-side dependency updates for Tauri, `clap`, `tar`, and `directories` are included in this release train.
- No manual workspace migration is expected, but teams that script habit-history imports should validate their MCP flows after upgrading.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `v2.2.0` is the recommended minor upgrade on top of `v2.1.x`.
- This release is focused on habit-history automation and a more reliable settings/sidebar recovery path.
