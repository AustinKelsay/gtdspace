# GTD Space {{tag}}

## Highlights

- Fixed a startup sidebar error caused by legacy `Purpose and Principles` paths being restored from persisted tabs instead of being normalized back to `Purpose & Principles`.
- Fixed the sidebar and tab actions for revealing a file in Finder or Explorer so they resolve the real on-disk path before launching the native file manager.

## Notable Changes

- Persisted tab restoration now canonicalizes the legacy Purpose section alias before the app reopens saved tabs or reuses the last active file path.
- The desktop `open_file_location` command now resolves existing paths first, then reveals files or opens directories using the correct OS-specific target on macOS, Windows, and Linux.
- Added regression coverage for both the sidebar alias reload path and the tab-persistence canonicalization path.
- This is a patch release focused on reliability and launch-time correctness; no workspace migration is required.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `v2.3.2` is the recommended patch upgrade for users who saw `Failed to load section files for .../Purpose and Principles` on launch.
- If you had a previously restored tab under the legacy Purpose alias, the app should now reopen it under the canonical `Purpose & Principles` path automatically.
