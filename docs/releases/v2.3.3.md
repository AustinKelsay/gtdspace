# GTD Space {{tag}}

## Highlights

- Fixed a remaining startup path issue where legacy `Purpose and Principles` aliases could still reach the app through non-tab startup flows and trigger a sidebar load failure.
- Path normalization now rewrites the legacy Purpose folder alias globally before sidebar, tab, and startup logic consume it.

## Notable Changes

- Moved legacy Purpose alias canonicalization into the shared path utility so all normalized app paths use `Purpose & Principles`.
- Added focused regression coverage for both forward-slash and Windows-style legacy paths.
- This is a patch release focused on startup reliability and path consistency; no workspace migration is required.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `v2.3.3` is the recommended patch upgrade if `v2.3.2` still showed `Failed to load section files for .../Purpose and Principles` on launch.
- Existing references to the legacy alias should now be normalized automatically anywhere the app runs shared path normalization.
