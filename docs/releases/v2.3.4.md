# GTD Space {{tag}}

## Highlights

- Fixed MCP action creation so `action_create.projectPath` must resolve to an existing GTD project instead of silently accepting a typo and targeting a stray folder path.
- Tightened the MCP tool contract and added end-to-end regression coverage for invalid project paths.

## Notable Changes

- `action_create` now validates that `projectPath` points to an existing project directory or project README before storing a change set.
- Unknown project paths now fail fast with a clear error instead of relying on later file writes that could create parent folders implicitly.
- Added workspace-level regression tests in the MCP backend plus a stdio integration test covering the exact bad request shape.
- This is a patch release focused on MCP write safety and clearer tool semantics; no workspace migration is required.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `v2.3.4` is the recommended patch upgrade if you use the GTD Space MCP server to create actions programmatically.
- MCP clients should send `projectPath` as either the project directory path or the project README path, using workspace-relative paths such as `Projects/Alpha Project` or `Projects/Alpha Project/README.md`.
