# GTD Space {{tag}}

## Highlights

- Hardened MCP write semantics so clients get clearer guidance when `action_create.projectPath` is wrong instead of silently drifting into an invalid target path.
- Exposed the GTD Space backend `serverVersion` through MCP so stale client binaries and mismatched installs are easier to diagnose.

## Notable Changes

- `workspace_info` and the generated workspace context now surface `serverVersion` from the GTD Space backend build.
- Unknown `action_create.projectPath` inputs now fail with explicit `workspace_list_items({ itemType: "project" })` guidance plus close project-path suggestions when there is a likely typo.
- Mutating MCP tool descriptions, context guidance, and settings copy now make the dry-run plus `change_apply` contract much harder to misread.
- Added backend, stdio, and UI regression coverage for typoed project paths and for planned changes that must not write until `change_apply`.
- No workspace migration is required for this release.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `v2.3.5` is the recommended patch upgrade if you use GTD Space through MCP clients that plan writes and apply them separately.
- This release improves MCP diagnostics and write safety, but it does not fix client-side tool-validation issues outside GTD Space itself.
