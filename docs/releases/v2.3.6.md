# GTD Space {{tag}}

## Highlights

- Made MCP change-set handling recoverable across the full dry-run/apply lifecycle so client retries no longer degrade into generic `Unknown change set` failures.
- Repeated `change_apply` for an already-applied plan now safely replays the original apply receipt within the same MCP server session instead of attempting duplicate writes.

## Notable Changes

- Pending change sets now move into bounded in-memory terminal tombstones when they are applied, discarded, invalidated by `workspace_refresh`, invalidated by revalidation, or fail during apply.
- `change_apply` now returns `replayed: true` when a client repeats an already-applied `changeSetId` in the same MCP server process and the server replays the stored receipt without re-running filesystem mutations.
- `workspace_refresh`, discarded plans, stale plans, and previously failed applies now return explicit terminal-state errors instead of collapsing to plain `Unknown change set`.
- Truly unknown change-set ids now include session/restart guidance so stale MCP sessions are easier to diagnose.
- Added backend and stdio regression coverage for apply replay, refresh invalidation, discarded plans, stale plans, and bounded tombstone eviction.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `v2.3.6` is the recommended patch upgrade if you use GTD Space through MCP clients that plan writes, retry `change_apply`, or refresh the workspace mid-session.
- Change-set tombstones remain session-scoped in memory. Restarting the MCP server still invalidates prior change-set ids.
