# GTD Space {{tag}}

## Highlights

- Added cache-backed Google Calendar MCP access so local MCP clients can inspect synced calendar events without making direct Google API calls.
- Fixed GTD sidebar action counts and project lists so completed and cancelled actions no longer inflate open-work totals.

## Notable Changes

- Added the `gtdspace://integrations/google-calendar/events.json` MCP resource plus the `google_calendar_list_events` tool with filtering for date bounds, query text, cancelled events, and result limits.
- Persisted Google Calendar sync data to an app-data cache file and hardened cache reads and writes so desktop sync and MCP consumers share the same on-disk source safely.
- Corrected all-day Google Calendar end-date handling so exclusive date-only end bounds no longer leak events into the following day.
- Improved the GTD workspace sidebar loading model so project action counts, empty states, and expanded project views stay aligned while actions refresh.
- Expanded Rust and frontend regression coverage for Google Calendar cache behavior, MCP stdio access, sidebar project loading, and open-action partitioning.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `v2.4.0` is the recommended upgrade if you use GTD Space with Google Calendar sync and want those events available through the standalone MCP server.
- Google Calendar MCP reads are cache-only in this release. OAuth and upstream sync still run through the desktop app, which refreshes the shared cache file.
