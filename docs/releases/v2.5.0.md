# GTD Space {{tag}}

## Highlights

- Added cache-backed Google Calendar MCP access so local MCP clients can inspect synced calendar events without making direct Google API calls.
- Hardened MCP workspace resolution, indexing, and dashboard habit/status behavior so cached context, pagination, and daily progress stay aligned.

## Notable Changes

- Added the `gtdspace://integrations/google-calendar/events.json` MCP resource plus the `google_calendar_list_events` tool with filtering for date bounds, query text, cancelled events, cursors, and result limits.
- Removed the legacy `gtdspace://item/*` MCP resource URIs from the `src-tauri/src/mcp_server.rs` resource list. Clients that still dereference those URIs should migrate to `workspace_get_item` for structured metadata, `workspace_read_markdown` for file contents, and `workspace_list_items` or `workspace_search` to rediscover item paths.
- Improved workspace path validation, fingerprint stability, README extension handling, and context-pack cache validation across the standalone MCP server backend.
- Tightened frontend MCP settings command quoting, workspace resolution, and habit-progress handling so launch commands, calendar history, and midnight-sensitive summaries stay accurate.

## Installation

Download the installer or package that matches your platform:

- Windows: `.msi` installer or `.exe` (NSIS) installer
- macOS Intel: `.dmg` for Intel Macs
- macOS Apple Silicon: `.dmg` for Apple Silicon Macs
- Linux: `.AppImage` (universal) or `.deb` (Debian/Ubuntu)

## Notes

- `{{tag}}` is the recommended upgrade if you use GTD Space with Google Calendar sync or the standalone MCP server.
- The standalone MCP server still reads Google Calendar data from the desktop app cache only; OAuth and upstream sync remain desktop-app responsibilities.
